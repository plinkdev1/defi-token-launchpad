# Regulatory & Compliance Framework for TRZ

## Safe-Harbor Declaration

**TRZ is not intended or marketed as a financial instrument.** It is a governance utility asset granting access to decentralized protocol operations. It provides no claim on assets, revenue, dividends, or profit. Users participate at their own risk and must comply with local jurisdictions.

---

## Governance-Safe Language Guide

### Prohibited Terms (Do Not Use)
The following terms imply TRZ is a financial investment and **must be avoided**:

| Prohibited | Reason |
|-----------|--------|
| "Investment" | Implies financial return expectation |
| "Returns" | Suggests guaranteed or expected profits |
| "Yield farming" | Implies passive income generation |
| "APY / APR" | Suggests interest or guaranteed rates |
| "Staking rewards" | Implies financial compensation |
| "Payouts" | Suggests dividend or profit distributions |
| "Passive income" | Implies financial gain without effort |
| "Appreciation" | Implies price-based financial benefit |
| "Price targets" | Implies investment performance claims |
| "Profit sharing" | Implies revenue allocation |
| "Interest" | Implies financial compensation |
| "Equity / Ownership" | Implies company ownership stake |

### Approved Terminology
Use these governance-focused terms instead:

| Approved | Context |
|----------|---------|
| "Governance participation" | Voting and proposal mechanisms |
| "Voting power" | Governance influence |
| "Discretionary allocation" | DAO-controlled token distribution |
| "Protocol coordination" | Governance decision-making |
| "Treasury management" | Community-controlled fund operations |
| "Parameter adjustments" | Protocol configuration via governance |
| "Escrowed TRZ (beTRZ)" | Vesting allocations from governance, not "rewards" |
| "Governance utility token" | TRZ's primary classification |
| "Non-custodial participation" | Users retain control, not financial delegation |
| "Vesting schedule" | Time-based token release (not "unlock" or "claiming rewards") |

## Jurisdiction Compliance

### Key Regulatory Principles
1. **Non-Security Classification**: TRZ is not a security under Howey Test
   - No investment contract (not expecting profits from effort of others)
   - No promise of returns from token price appreciation
   - Purely governance utility

2. **Non-Investment Product**: Complies with investment regulation exemptions
   - DAO governance tokens may qualify for safe harbor
   - Utility focus prevents investment classification
   - Discretionary allocation removes ROI expectation

3. **User Responsibility**: Participants acknowledge:
   - Understanding of blockchain technology risks
   - Compliance with local laws
   - No reliance on financial predictions
   - Voluntary participation

### Geographic Considerations
- **CFTC (USA)**: TRZ governance token, not a commodity future
- **SEC (USA)**: Governance utility, not a security
- **ESMA (EU)**: Utility token under MiCA framework
- **Singapore**: Utility token classification under Payment Services Act
